# Настройка Mattermost MCP

> **Это инструкция ДЛЯ AI-агента, не для человека.**
> Если пользователь попросил настроить mattermost-mcp — следуй шагам ниже.

## Что это

MCP-сервер для Mattermost. Устанавливается через `npx github:runotwo/mattermost-mcp`. Требует env-переменные для авторизации.

## Правила

- Выполняй шаги **строго по порядку**
- На каждом шаге **задавай вопрос и жди ответа** — не пропускай и не додумывай
- Общайся с пользователем **на русском**
- Предлагай варианты **нумерованным списком** — пользователь отвечает цифрой
- Не выводи пароли и токены в текст ответа

## Defaults

- `MM_URL` = `https://blinkmap.loop.ru` (не спрашивай URL, используй этот)

---

## Шаг 1: Способ авторизации

Спроси у пользователя (ровно этот текст):

```
Подключаю Mattermost MCP. Выбери способ авторизации:

1. Personal Access Token — один токен, проще всего
2. Логин + пароль + TOTP — если токены не включены на сервере
```

Жди ответа (цифру). Не продолжай, пока не ответит.

---

## Шаг 2A: Personal Access Token

Если выбрал **1**:

Спроси:

```
У тебя уже есть Personal Access Token?

1. Да, есть
2. Нет, нужно создать
```

Жди ответа.

### Если нужно создать (ответил 2):

Скажи:

```
Как создать Personal Access Token:

1. Открой Loop в браузере (https://blinkmap.loop.ru)
2. Аватарка (левый верхний угол) → Profile
3. Security (в левом меню)
4. Прокрути до Personal Access Tokens → Create Token
5. Описание: mcp-server → Save
6. Скопируй Access Token — он показывается только один раз

Если секции Personal Access Tokens нет — она выключена. Скажи, пойдём через логин+пароль.
```

Жди, пока пришлёт токен. Сохрани как `MM_TOKEN`.

Иди на **Шаг 3**.

### Если уже есть (ответил 1):

Скажи: "Скинь токен."

Жди токен. Сохрани как `MM_TOKEN`.

Иди на **Шаг 3**.

---

## Шаг 2B: Логин + пароль + TOTP

Если выбрал **2**:

### Логин и пароль

Скажи: "Скинь логин и пароль от Loop (те, что на странице входа)."

Жди. Сохрани как `MM_USERNAME` и `MM_PASSWORD`.

### TOTP-секрет

Спроси:

```
Теперь нужен TOTP-секрет — base32-строка, из которой генерируются 6-значные коды двухфакторки.
Это НЕ сам код из Google Authenticator, а строка вроде JBSWY3DPEHPK3PXP.

1. Есть, сейчас скину
2. Нет MFA (двухфакторка выключена)
3. Потерян (сканировал QR и не сохранил)
```

Жди ответа.

**Ответ 1** → жди секрет, сохрани как `MM_TOTP_SECRET`. Иди на Шаг 3.

**Ответ 2** → не ставь `MM_TOTP_SECRET`. Иди на Шаг 3.

**Ответ 3** → скажи:

```
Из Google Authenticator секрет не достать. Нужно пересоздать:

1. Открой Loop → Аватарка → Profile → Security
2. Multi-factor Authentication → Remove MFA (ввести текущий 6-значный код)
3. Add MFA заново
4. Под QR-кодом будет текстовая строка — это TOTP-секрет
   (выглядит как IIWDLAPY6ONL4WWMTFZYXS3Q3AHBBN6Q)
5. Скопируй и скинь мне эту строку
6. Отсканируй QR в Google Authenticator заново
7. Подтверди 6-значным кодом
```

Жди секрет. Сохрани как `MM_TOTP_SECRET`. Иди на **Шаг 3**.

## Шаг 3: Куда добавить

Спроси:

```
Куда добавить Mattermost MCP?

1. В текущий проект (.mcp.json в корне)
2. Глобально (во все проекты)
```

Жди ответа.

- **1** → конфиг-файл: `.mcp.json` в текущей рабочей директории
- **2** → конфиг-файл: `~/.claude/settings.json` (ключ `mcpServers`). Для Cursor: `~/.cursor/mcp.json`

---

## Шаг 4: Запиши конфиг

Собери блок:

```json
{
  "mattermost": {
    "command": "npx",
    "args": ["-y", "github:runotwo/mattermost-mcp"],
    "env": {
      "MM_URL": "https://blinkmap.loop.ru"
    }
  }
}
```

Добавь в `env`:
- **PAT**: `"MM_TOKEN": "<токен>"`
- **Логин**: `"MM_USERNAME": "<логин>"`, `"MM_PASSWORD": "<пароль>"`, и `"MM_TOTP_SECRET": "<секрет>"` (если есть)

Прочитай существующий конфиг-файл. Добавь `"mattermost"` внутрь `"mcpServers"`. Не перетирай другие серверы. Если файла нет — создай с обёрткой `{ "mcpServers": { ... } }`.

Не печатай пароли и токены в текстовый ответ пользователю — просто молча запиши файл.

## Шаг 5: Проверь подключение

Запусти:

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}' | MM_URL=https://blinkmap.loop.ru MM_TOKEN=<token> npx -y github:runotwo/mattermost-mcp 2>/dev/null | head -1
```

Для логин-режима используй `MM_USERNAME`, `MM_PASSWORD`, `MM_TOTP_SECRET` вместо `MM_TOKEN`.

Подставь реальные значения.

**Ожидаемый результат**: JSON с `"serverInfo":{"name":"mattermost","version":"1.0.0"}`.

Если не работает — скажи пользователю, что пошло не так, предложи повторить.
